# ПРОЕКТ: РЕДИЗАЙН САЙТА НАТК (ДИПЛОМНЫЙ ПРОЕКТ)

## 🛠 ТЕХНОЛОГИЧЕСКИЙ СТЕК
- **Языки программирования:** TypeScript (основной), SQL.
- **Backend:**
  - NestJS (фреймворк)
  - PostgreSQL (база данных)
  - TypeORM (ORM для работы с БД)
  - Axios + Cheerio (инструменты для парсинга HTML)
  - NestJS Schedule (Cron задачи для автоматического обновления)
- **Frontend:**
  - Next.js 15 (App Router)
  - Tailwind CSS (стилизация)
  - Lucide React (иконки)
  - TypeScript

## 📊 БАЗА ДАННЫХ (PostgreSQL в 3NF)
Создано 15 взаимосвязанных таблиц:
- `news`, `news_images` (Новости и их галереи)
- `staff`, `staff_positions` (Преподаватели, руководство и их должности)
- `documents`, `document_categories` (Документы колледжа)
- `companies`, `vacancies` (Работодатели и вакансии для студентов)
- `campuses`, `specialties`, `groups` (Корпуса, специальности и учебные группы)
- `subjects`, `rooms`, `schedules` (Дисциплины, аудитории и расписание занятий)
- `departments` (Структурные подразделения колледжа)

## 🕵️ BACKEND: ПАРСЕР И API
- **Умный парсер (ParserService):**
  - Сбор новостей с датами и изображениями.
  - Сбор руководства и педагогического состава (с фото).
  - Сбор официальных документов (поддержка пагинации Joomla, фильтрация мусора, умная очистка названий).
  - Сбор вакансий (автоматическая привязка к компаниям).
  - Сбор структуры (парсинг иерархии h3/h4, группировка по категориям Минобрнауки).
  - Сбор расписания (автоматическое определение специальности группы через страницы расписания).
- **Manual Sync API:**
  - Точка доступа `POST /api/sync` для ручного запуска парсера.
  - Поддержка параметров (например, `?type=structure`), чтобы обновлять данные выборочно.
- **REST API:**
  - Созданы контроллеры для всех сущностей (новости, расписание, сотрудники и т.д.).
  - Сортировка новостей по дате (свежие сверху).
  - Фильтрация расписания по группам.
  - Включен CORS для связи с фронтендом.

## 🎨 FRONTEND: ИНТЕРФЕЙС
- **Система ролей:** Выбор между "Студентом" и "Абитуриентом" (сохраняется в localStorage).
- **Личный кабинет студента (Dashboard):**
  - Приветственный баннер.
  - Быстрые ссылки.
  - "Мини-расписание": отображение пар только на сегодня и завтра для удобства.
- **Новости:** Сетка новостей с модальными окнами для чтения контента.
- **Расписание:**
  - Выбор группы студента (сохраняется в браузере).
  - Группировка по дням недели.
  - Поддержка подгрупп.
- **О колледже:** Страница с подробной информацией (из статического JSON).
- **Структура:** 
  - Иерархическое отображение по категориям (Руководство, Органы управления, Учебно-методическая деятельность и т.д.).
  - Карточки подразделений с контактами и руководителями.

## ✅ ТЕКУЩИЙ СТАТУС
- База данных полностью спроектирована и подключена.
- Парсер настроен на этичный сбор данных с задержками (sleep).
- Фронтенд и бэкенд полностью связаны через API.
