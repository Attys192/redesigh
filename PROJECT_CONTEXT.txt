# ПОДРОБНЫЙ КОНТЕКСТ ПРОЕКТА: РЕДИЗАЙН САЙТА НАТК (ДИПЛОМНЫЙ ПРОЕКТ)

Этот файл содержит полное описание структуры, технологического стека и текущего прогресса проекта для понимания контекста ИИ-ассистентом (Gemini).

---

## 🏗 ОБЩАЯ АРХИТЕКТУРА
Проект представляет собой современное веб-приложение, разделенное на две основные части:
1. **Backend (`natk-backend`)**: API-сервер на NestJS, отвечающий за сбор данных (парсинг) и их предоставление фронтенду.
2. **Frontend (`natk-frontend`)**: Клиентское приложение на Next.js 15, предоставляющее интерфейс для студентов и абитуриентов.

---

## 🛠 ТЕХНОЛОГИЧЕСКИЙ СТЕК

### **Backend (Серверная часть)**
- **Framework**: NestJS (TypeScript).
- **Database**: PostgreSQL (через TypeORM).
- **Scraping**: Axios + Cheerio (этичный сбор данных с задержками `sleep`).
- **Scheduling**: @nestjs/schedule (Cron-задачи для автообновления данных).
- **API**: REST API с поддержкой CORS для взаимодействия с фронтендом.
- **Особенности**: 
  - Обработка HTTPS-сертификатов (игнорирование ошибок сертификата Минцифры для доступа к natk.ru).
  - Глубокий парсинг (не только списки, но и вложенные страницы новостей, профили преподавателей, галереи изображений).

### **Frontend (Клиентская часть)**
- **Framework**: Next.js 15 (App Router, React 19).
- **Styling**: Tailwind CSS 4 (современная стилизация).
- **Icons**: Lucide React.
- **State**: React Hooks + localStorage (сохранение роли пользователя и выбранной группы).
- **Особенности**:
  - Адаптивный дизайн (Mobile-first).
  - Разделение путей для студентов (`/student`) и абитуриентов (`/applicant`).
  - Система модальных окон для просмотра контента без перезагрузки.

---

## 📊 СТРУКТУРА ДАННЫХ (БАЗА ДАННЫХ)
Реализована реляционная схема в 3-й нормальной форме (3NF) из 15 таблиц:
- **Новости**: `news`, `news_images` (поддержка галерей).
- **Сотрудники**: `staff`, `staff_positions` (руководство, педагоги, био, достижения).
- **Документы**: `documents`, `document_categories` (фильтрация мусора, категории).
- **Карьера**: `companies`, `vacancies` (связка вакансий с работодателями).
- **Учеба**: `campuses`, `specialties`, `groups` (корпуса, коды специальностей, группы).
- **Расписание**: `subjects`, `rooms`, `schedules` (дисциплины, аудитории, расписание с поддержкой подгрупп).
- **Структура**: `departments` (иерархия подразделений колледжа).

## 🔍 ФУНКЦИОНАЛЬНЫЕ ВОЗМОЖНОСТИ

### **Умный Парсер (ParserService)**
- **Новости**: Собирает заголовки, даты, HTML-контент и все изображения из галерей (sigplus).
- **Сотрудники**: Извлекает ФИО, должность, фото, био и ссылки на дипломы/достижения.
- **Расписание**: Автоматически определяет специальность группы при парсинге страниц расписания.
- **Синхронизация**: Точка `POST /api/sync?type=...` позволяет обновлять данные выборочно (news, staff, schedule и т.д.).

### **Интерфейс Студента**
- **Dashboard**: Приветствие, текущая группа, мини-расписание на "Сегодня" и "Завтра".
- **Полное расписание**: Фильтрация по группе, отображение по дням недели.
- **Вакансии**: Список актуальных предложений от работодателей-партнеров.
- **Документы**: Удобный доступ к официальным документам колледжа.

### **Интерфейс Абитуриента**
- **Специальности**: Список направлений с проходными баллами (парсятся автоматически).
- **Калькулятор**: Инструмент для расчета шансов на поступление (в разработке).
- **Тест**: Профориентационный тест для выбора специальности.

## 📁 СТРУКТУРА ФАЙЛОВ ПРОЕКТА

### **Backend (`natk-backend/src`)**
- `api/`: Контроллеры и сервисы для предоставления данных фронтенду.
- `entities/`: Определение схем таблиц TypeORM.
- `parser/`: Логика скрейпинга данных с natk.ru.
- `!!!specialties.data.ts`: Статический справочник специальностей и корпусов для сидирования.

### **Frontend (`natk-frontend/src`)**
- `app/`: Основные страницы (App Router).
  - `student/`: Личный кабинет и сервисы для студентов.
  - `applicant/`: Раздел для поступающих.
- `components/`: Общие UI-компоненты (модалки, ссылки и т.д.).
- `types/`: Общие TypeScript интерфейсы.

---

## ✅ ТЕКУЩИЙ СТАТУС РАЗРАБОТКИ
1. **Бэкенд**: Полностью готов к работе. Парсер настроен и наполняет БД реальными данными.
2. **Фронтенд**: Реализованы основные страницы. Настроена связь с API.
3. **Данные**: БД наполнена актуальной информацией о новостях, сотрудниках и специальностях.
4. **Интеграция**: Фронтенд успешно получает и отображает данные из локального API (localhost:3000).

## 🚀 КАК ЗАПУСТИТЬ

### **Backend**
1. `cd natk-backend`
2. `npm install`
3. Настроить PostgreSQL (БД: `natk_db`)
4. `npm run start:dev`
5. Для первичного наполнения: `POST http://localhost:3000/api/sync`

### **Frontend**
1. `cd natk-frontend`
2. `npm install`
3. `npm run dev` (запуск на localhost:3001)
